# Output — Landing Page (tipo `lp`)

A estrutura da LP não é template — é definida estrategicamente com base no produto, no ICP e no estágio de consciência do visitante. Seções possíveis (usar as que fazem sentido, na ordem que converte):

```
HERO
Headline: [promessa central — clara, específica, sem genérico]
Subheadline: [mecanismo + proof point + eliminação de objeção]
CTA: [ação específica]

SEÇÃO DE CREDIBILIDADE (se há proof points fortes)
[números, logos, depoimentos — só se reais]

PROBLEMA / DIAGNÓSTICO
[nomear a dor do ICP com precisão — o que ele perde, teme ou quer evitar]

SOLUÇÃO / MECANISMO
[como o produto resolve — mecanismo real, não vago]

COMO FUNCIONA (se o produto tem processo)
[etapas claras — máximo 4]

BENEFÍCIOS
[resultados concretos, não features — o que muda na vida/negócio do ICP]

OBJEÇÕES
[as 2-3 objeções principais do ICP, respondidas]

PROVA SOCIAL
[depoimentos com nome, cargo, resultado específico]

FAQ
[perguntas reais que o ICP tem antes de converter]

CTA FINAL
[repetir ação + elemento de urgência ou garantia se disponível]
```

**Regras para LP:**
- Nunca usar seção sem conteúdo real para preencher
- Diagnóstico antes de prescrição — nomear o problema antes de vender a solução
- Dos and don'ts do stakeholder aplicados em cada seção
- Copy de cada seção com headline + parágrafo (não bullet list seca)
- Se o stakeholder tem preferências de linguagem documentadas, espelhar
